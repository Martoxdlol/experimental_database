Domain-aware modules (key encoding, version resolution, document insert/get with MVCC) belong in higher layers.

LAYER 2 INDEPENDENCE TEST: Layer 2 (Storage Engine) MUST be usable as a standalone generic storage library with no document/MVCC knowledge. Its public facade should look like:
  - StorageEngine::open/close — lifecycle
  - StorageEngine::create_btree() → BTreeHandle — allocates a root page, returns a handle
  - StorageEngine::open_btree(root_page) → BTreeHandle — opens existing tree
  - BTreeHandle::get/insert/delete(key: &[u8], value: &[u8]) — raw byte key/value ops
  - BTreeHandle::scan(lower, upper, direction) → ScanIterator — range scan on bytes
